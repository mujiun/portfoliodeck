Principles

These rules keep the experience usable across offerings, while speeding growth and lowering cost

Conditional setup vs. additional setup

Use conditional setup when an action changes the message format itself — like showing a catalog preview or a button that opens a flow. If the action doesn't affect the format, use additional setup instead. Owner teams can book office hours to customize a conditional setup component and have flexibility to evolve it as the feature grows.

Pre-select an option when there are multiple

On the landing state, default to one option rather than leaving everything unselected. Pick the most common use case, or the path you want users to take.

This is recommended as the easiest option for your customers. Zero-tap will automatically send code without requiring your customer to tap a button. An autofill or copy code message will be sent if zero-tap and autofill aren't possible.

By selecting zero-tap, I understand that Coon Buffet's use of zero-tap authentication is subject to the WhatsApp Business Terms of Service. It's Coon Buffet's responsibility to ensure its customers expect that the code will be automatically filled-in on their behalf when they choose to receive the zero-tap code through WhatsApp.Learn more and review best practices.

Governance

Guidelines for evolving the system responsibly and getting support when you need it

When to add a new message type vs. expand an existing one

Expand an existing type

  • The goal and core flow match an existing message type
  • The need can be met by adding a variant or setting
  • Users would expect it to live alongside the current type
  • It reuses the same shared components and sequence

Add a new message type

  • It serves a distinct goal or audience
  • The flow or required components differ meaningfully
  • Expanding an existing type would overload or confuse it
  • It needs its own template and configuration sequence
